Overview

What is ATB Mobile?

A production-style banking application built from scratch in Flutter, so a single codebase runs on both Android and iOS. It is wrapped in the bank's official burgundy brand identity and works even with no internet connection.

🎯 The goal

Help ATB customers find the nearest ATM/agence, get instant answers about banking services, and manage their cards — all from one app.

🧩 What makes it different

Real ATM data from OpenStreetMap, an AI assistant fluent in Tunisian dialect, and downloadable offline maps.

⚙️ How it's built

Clean separation of models, services and screens. No backend required — it talks directly to public APIs.

🛟 Resilient by design — the fallback

If GPS is denied or the Overpass API is slow/unreachable, the app instantly falls back to 20 hand-curated Tunisian ATMs (ATB, STB, BNA, BIAT, Zitouna, UIB, Amen…). The map is never empty, even with no network.

Feature 4 · the hard part

📱 Offline maps with MBTiles

  • Downloads every map tile for a region and stores them in a single SQLite (.mbtiles) file.
  • Resumable: already-downloaded tiles are skipped, so an interrupted download just continues.
  • Respects OSM rate limits (50 ms between tiles) and a polite custom User-Agent.
  • Live progress bar with % and tile count; cancellable mid-download.
  • A custom MBTilesTileProvider bridges SQLite → flutter_map so the map renders with zero internet.

Pre-configured regions

RegionZoomEst. size
Grand Tunis10–15~90 MB
Sfax10–15~45 MB
All Tunisia5–11~80 MB

Tile counts are computed with slippy-map math; rows stored in TMS order. Stored in the app's private docs folder — no storage permission needed.

Engineering

Clean, layered architecture

lib/
├─ config/     secrets (gitignored) + example
├─ models/     ATM · ChatMessage
├─ services/   business logic & APIs
│   ├─ atm_service        // Overpass + fallback
│   ├─ chat_service       // Groq LLaMA
│   ├─ location_service   // GPS + distance
│   ├─ offline_map_service// MBTiles
│   └─ mbtiles_provider   // SQLite → map
├─ screens/    home · map · cartes · messages
│              profil · assistance · atm_detail
├─ theme.dart   // brand colors
└─ main.dart    // entry point

Why this structure

  • Models = pure data (with an Overpass JSON parser).
  • Services = all I/O and logic, fully testable and UI-free.
  • Screens = presentation only; they call services.
  • API keys live in a gitignored secrets.dart with a committed template — safe to open-source.

Technical highlights

Engineering decisions I'm proud of

🛟 Never breaks

Every external call (GPS, Overpass, Groq) has a graceful fallback — curated ATMs, mock AI replies, cached tiles. The app stays usable offline or when an API is down.

🔁 Resumable downloads

Tile downloads checkpoint at the tile level in SQLite, so a dropped connection costs you nothing — it picks up exactly where it stopped.

🧠 Context-aware AI

The assistant isn't generic — it's fed the user's actual nearby ATMs and recent conversation, so answers are specific and grounded.

🔐 Safe to open-source

Secrets are isolated and gitignored with a committed template, so contributors can run it without ever leaking a key.
